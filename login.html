🔥 Set Up Real Google Login (Firebase)

Follow these 7 steps to enable real Google login on ShadowPath. It's completely free and takes about 10 minutes. No coding required — just copy and paste.

1

Create a Firebase Project

Go to console.firebase.google.com. Sign in with your Google account if prompted.

Click the big blue "Add project" button. Then:

a Type shadowpath as the project name and click Continue
b On the "Google Analytics" screen — you can leave it enabled or turn it off, doesn't matter — then click Continue or Create project
c Wait for the spinner to finish, then click "Continue" to enter your new project dashboard
2

Enable Google & Email Sign-In

In the left sidebar of the Firebase console, look for Authentication (it has a person icon). Click it.

a Click the "Get started" button if it appears
b Click the "Sign-in method" tab at the top
c Click "Google" in the list → toggle the "Enable" switch to on (turns blue) → pick your email as the project support email → click "Save"
d Click "Email/Password" in the list → toggle the first "Enable" switch to on → click "Save"
✅ You should now see both Google and Email/Password showing as Enabled in the list.
3

Register Your Web App & Get the Config

Now you need to tell Firebase that ShadowPath is a web app so it gives you the connection keys.

a Look at the very top-left of the Firebase console. Click the ⚙️ gear icon right next to "Project Overview" in the left sidebar. Then click "Project settings"
b On the Project Settings page, scroll down until you see a section called "Your apps". It might say "No apps yet"
c Click the </> icon (the web icon — it looks like a pair of angle brackets). If you also see Android and iOS icons, make sure you click the web one specifically
d In the "App nickname" box type shadowpath-web (or anything you like). Do NOT check the "Firebase Hosting" checkbox — leave it unchecked. Click "Register app"
e Firebase now shows you a code block that starts with const firebaseConfig = {. You need to copy everything inside the curly braces. It will look like the box below — yours will have real values instead of the placeholders
const firebaseConfig = { apiKey: "AIzaSyD-abc123xyz...", authDomain: "shadowpath-abc12.firebaseapp.com", projectId: "shadowpath-abc12", storageBucket: "shadowpath-abc12.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abc123def456" };

Select all of that code block and copy it (Ctrl+C on Windows, Cmd+C on Mac). Then click "Continue to console" to dismiss the screen.

💡 You can always get back to this config: Project Settings → Your apps → SDK setup and configuration → select "Config"
4

Open auth.js and Paste Your Config

Now you need to put that config code into the auth.js file inside your shadowpath folder.

a Open your C:\Users\janme\shadowpath folder in File Explorer
b Right-click on auth.js → hover over "Open with" → choose Notepad (or VS Code if you have it)
c Near the very top of the file, find this block — it starts on about line 16:
const FIREBASE_CONFIG = { apiKey: "REPLACE_WITH_YOUR_API_KEY", authDomain: "REPLACE_WITH_YOUR_AUTH_DOMAIN", projectId: "REPLACE_WITH_YOUR_PROJECT_ID", storageBucket: "REPLACE_WITH_YOUR_STORAGE_BUCKET", messagingSenderId: "REPLACE_WITH_YOUR_MESSAGING_SENDER_ID", appId: "REPLACE_WITH_YOUR_APP_ID", };
d Select everything from const FIREBASE_CONFIG = { all the way to the closing }; and delete it
e Paste the config you copied from Firebase in its place. Change firebaseConfig to FIREBASE_CONFIG — just the variable name at the start needs to match
f Save the file: Ctrl+S
✅ After saving, the line should start with const FIREBASE_CONFIG = { and the values inside should be your real keys — not the REPLACE_WITH_... placeholders.
5

Run the Site with VS Code Live Server (Required for Google Login)

Google login will not work if you just double-click index.html to open it. You must run it on a local web server. The easiest way is VS Code + Live Server.

a Download VS Code if you don't have it (free). Install it and open it
b In VS Code, click the Extensions icon in the left sidebar (looks like 4 squares). Search for "Live Server" by Ritwick Dey. Click Install
c Click File → Open Folder and open your C:\Users\janme\shadowpath folder
d Click on login.html in the file list to open it. Then look at the very bottom-right corner of VS Code — you'll see a button that says "Go Live". Click it
e Your browser should automatically open to http://127.0.0.1:5500/login.html — this is your site running on a local server ✅
💡 Alternatively, if you have Node.js installed you can open a terminal in the shadowpath folder and run: npx serve . then open http://localhost:3000
6

Add localhost to Firebase Authorized Domains

Firebase needs to know your site's address is allowed to use Google login. Since you're running it locally, you need to add localhost and 127.0.0.1.

a Go back to the Firebase console → Authentication in the left sidebar
b Click the "Settings" tab (next to "Users" and "Sign-in method")
c Scroll down to "Authorized domains". You'll see localhost might already be listed — if so, you're good! If not, click "Add domain" and type localhost, then click Add
d Also add 127.0.0.1 the same way if it's not already listed
🌐 When you publish your site to the internet (e.g. yoursite.com), you'll need to add that domain here too.
7

Test It — You're Done!

Go to your browser where Live Server opened the site (http://127.0.0.1:5500/login.html) and try signing in.

a Click "Continue with Google" — a real Google account picker popup should appear from Google. If you only have one Google account it will sign in automatically
b Or try the "Create Account" tab — fill in your name, email, and a password to create a real account with email/password login
c After signing in, you'll land on the ShadowPath app with your real Google name and profile
✅ The yellow "Demo Mode" banner will be gone once Firebase is configured correctly — that's how you know it's working!
🔴 Common errors and fixes:
? "auth/unauthorized-domain" → You haven't added localhost to Authorized Domains yet (Step 6)
? "auth/invalid-api-key" → The apiKey in auth.js is still the placeholder or was pasted incorrectly. Re-check Step 4
? Popup closes with no error → Make sure you're opening the site via Live Server (http://...) not as a file (file://...)
? Google button does nothing → Check the browser console (F12 → Console) for the exact error message
🚀 Want to put ShadowPath online? You can host it for free on Firebase Hosting — run npm install -g firebase-tools then firebase deploy. Or drag the shadowpath folder into Netlify for instant free hosting.
